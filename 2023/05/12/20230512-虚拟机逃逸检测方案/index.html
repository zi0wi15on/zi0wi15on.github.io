虚拟机逃逸检测方案

  • VMware ESXi
    • CVE-2018-6981: Arbitrary address free in vmxnet3 virtual ethernet card
    • CVE-2018-6982: Uninitialized stack variable in vmxnet3_cmd_get_coalesce()
  • Hyper-V Server
  • Xen
  • Hyper-V
    • CVE-2021-28476 : Hyper-V 远程代码执行漏洞
  • VMware Workstation
    • CVE-2017-4901: OOB in drag-and-drop (DnD) function
    • CVE-2017-4905: Uninitialized memory usage in drag-and-drop (DnD) function
    • CVE-2020-3947 : VMware Workstation/Fusion vmnetdhcp 组件UAF漏洞
    • CVE-2017-4934 : VMware Workstation/VMware Fusion vmnat组件堆溢出漏洞
  • VirtualBox
    • CVE-2018-2860: Oracle Virtualbox HGCM OOB Read/Write
    • CVE-2021-2119: OOB Read/Write in /src/VBox/Devices/Storage/VBoxSCSI.cpp: vboxscsiReadString() in VirtualBox 6.1.2
  • QEMU/KVM
    • CVE-2019-6778: Heap buffer overflow in /slirp/tcp_subr.c: tcp_emu()
    • CVE-2020-14364: OOB in /hw/usb/core.c: do_token_setup()

漏洞原理

不论是 Type-1 还是 Type-2 Hypervisor，虚拟机逃逸主要利用存在于 硬件虚拟化、网络协议虚拟化 等功能代码中的逻辑漏洞或内存破坏漏洞，在虚拟机相关进程或驱动中执行任意代码，进一步访问控制宿主机内资源，达到逃逸效果

检测方案

  1. 虚拟机内安装 agent
    1. 编写 yara 规则，检测已知漏洞利用代码（已经具备检测能力，待添加规则）
    2. 编写 yara 规则，检测漏洞利用程序用到的 shellcode，如 msf-shellcode-linux、msf-shellcode-windows 规则等（已经具备检测能力，已经有规则）
  2. 宿主机内安装 agent
    1. 实时监控虚拟机相关进程（见表）的行为，包括创建子进程、读写文件、发送网络请求等，为减少误报，可考虑初步只监控创建子进程行为，若目标进程存在创建子进程的行为，且子进程不在白名单内（见表），则判断为虚拟机逃逸，告警内容包括虚拟机产品名称、虚拟机进程名称、子进程启动完整参数等
