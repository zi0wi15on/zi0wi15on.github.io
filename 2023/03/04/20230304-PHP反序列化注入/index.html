介绍

为什么会有序列化

为了解决 PHP 对象传递的一个问题，PHP文件在执行结束以后就会将对象销毁，那么如果下次有一个页面恰好要用到刚刚销毁的对象就会束手无策；但是又不能永远不让它被销毁，于是人们就想出了一种能长久保存对象的方法，这就是 PHP 的序列化，当我们下次要用的时候只要反序列化一下就可

实现过程

函数

php 将数据序列化和反序列化会用到两个函数

  • serialize() 将对象格式化成有序的字符串

  • unserialize() 将字符串还原成原来的对象

序列化是以整个对象为单位进行传输, 而序列化一个对象将会保存对象的所有变量，但不会保存对象的方法，只会保存类的名字

如果了解底层的同学可以知道，类中的方法本就不在类中

public、protected、private

PHP 对属性或方法的访问控制，是通过在前面添加修饰符来实现的

  • public：公有的类成员可以在任何地方被访问

  • protected：受保护的类成员则可以被其自身以及其子类和父类访问

  • private：私有的类成员则只能被其定义所在的类访问（有可能是父类定义，子类继承但无法访问，此时序列化时标注的类名为父类）

不同修饰符序列化后的值不一样

  • public：属性被序列化的时候属性值会变成属性名

  • protected：属性被序列化的时候属性值会变成\x00*\x00属性名

  • private：属性被序列化的时候属性值会变成\x00类名\x00属性名

其中：\x00表示空字符，但是还是占用一个字符位置，这个在对序列化后的字符串尝试做修改的时候尤为重要

反序列化漏洞

序列化和反序列化本身没有问题，但是如果反序列化的内容是用户可以控制的,且后台不正当的使用了PHP中的魔法函数，就会导致安全问题

常见的，当程序在进行反序列化时，会调用一些函数，但是如果传入函数的参数可以被用户控制的话，用户可以输入一些恶意代码到函数中，从而导致反序列化漏洞

魔术方法

魔术方法是一种特殊的方法，当对对象执行某些操作时会覆盖 PHP 的默认操作

PHP 将所有以__（两个下划线）开头的类方法保留为魔术方法。 因此，除非覆盖 PHP 的行为，否则不建议使用此类方法名称

常见的魔术方法 触发条件
__construct() 类的构造函数，对象创建时触发，但unserialize()时不会自动调用
__destruct() 类的析构函数，对象销毁前触发
__sleep() 对象被序列化前触发
__wakeup() 在反序列化后触发
__call() 调用类不可访问或不存在的方法时触发
__callStatic() 调用类不可访问或不存在的静态方法时触发
__get() 调用类不可访问或不存在的属性时触发
__set() 设置类不可访问或不存在的属性时触发
__isset() 当对不可访问的属性调用isset()或empty()时触发
__unset() 当对不可访问的属性调用unset()时触发
__toString() 把类当作字符串使用时触发
__invoke() 将对象调用为函数时触发
__autoload() 调用不存在的类时触发
__clone() 对象复制完成时触发

特别说明

__toString() 的触发条件比较多，也因为这个原因容易被忽略，以下为常见的触发条件

  • echo($obj) 或 print($obj)，即输出对象时会触发

  • 反序列化对象与字符串连接时（将对象的类型转换为字符串）

  • 反序列化对象参与格式化字符串时（将对象的类型转换为字符串）

  • 反序列化对象与字符串使用==比较时（PHP进行==比较时会转换参数类型）

  • 反序列化对象参与格式化SQL语句时，一般为绑定参数操作

  • 反序列化对象在经过php字符串函数时，如 strlen()、addslashes()

  • 在in_array()方法中，第一个参数是反序列化对象，第二个参数的数组中有toString返回的字符串时

  • 反序列化的对象作为 class_exists() 的参数时
